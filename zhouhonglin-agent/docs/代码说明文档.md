## 项目概述

**ZhouHongLin 电商智能客服 Agent 系统** 是基于 RAG（检索增强生成）技术与 LangGraph Agent 框架构建的智能化客服解决方案。系统支持多平台订单同步、商品知识库管理、智能问答、物流渠道匹配等核心功能。

### 核心特性

| 特性 | 说明 |
|------|------|
| **RAG 检索增强** | 向量检索 + BM25 混合检索，支持文档切片、Embedding、重排序 |
| **多 Agent 架构** | Simple/Tool/RAG/Prompt Chaining/Routing/Reflection/Planning/Memory/Multi-Agent |
| **可视化管理** | 一体化 Web 界面，支持文档上传、知识库管理、参数配置 |
| **多模型兼容** | 支持 DeepSeek/Qwen/GLM-4 等主流大模型 |
| **工具扩展** | 开放的工具注册机制，支持自定义业务工具接入 |

## 核心模块说明

### 1. Web 应用模块 (`web_app.py`)

**职责**：提供 FastAPI 服务，处理 HTTP 请求，连接前端与后端核心逻辑。

**主要功能**：
- 静态文件服务（前端页面）
- 对话 API（流式响应）
- RAG 文档上传/检索/管理 API
- Agent 配置与状态管理

**关键代码结构**：
```python

# 静态文件挂载
```

## API 接口文档

### 1. 对话接口

#### POST /api/chat

**描述**：与 AI 客服进行对话，支持流式响应

**请求参数**：
```json
{
    "message": "你的问题",
    "collection_name": "资料库名称",
    "use_rag": true,
    "stream": true,
    "temperature": 0.7,
    "max_tokens": 2048
}
```

**响应**：SSE 流式响应，包含 token 和完整消息

### 2. RAG 文档上传

#### POST /api/rag/upload/files

**描述**：上传文件到指定资料库

**请求参数**：
- `files`: UploadFile[] - 文件列表
- `collection_name`: str - 目标资料库名称

**响应**：
```json
{
    "success": true,
    "message": "文件上传成功",
    "document_count": 5,
    "chunk_count": 20
}
```

#### POST /api/rag/upload/text

**描述**：上传文本内容到资料库

**请求参数**：
```json
{
    "text": "文档内容",
    "collection_name": "资料库名称",
    "source": "文档来源"
}
```

### 3. 资料库管理

#### GET /api/rag/collections

**描述**：获取所有资料库列表

**响应**：
```json
{
    "success": true,
    "collections": ["default", "product_knowledge", "after_sales"]
}
```

#### POST /api/rag/collection

**描述**：创建新的资料库

**请求参数**：
```json
{
    "name": "新资料库名称"
}
```

#### DELETE /api/rag/collection/{name}

**描述**：清空并删除指定资料库

**响应**：
```json
{
    "success": true,
    "message": "资料库已清空"
}
```

### 4. 文档查询

#### GET /api/rag/documents

**描述**：获取指定资料库的文档列表

**请求参数**：
- `collection_name`: str - 资料库名称
- `page`: int - 页码（默认 1）
- `page_size`: int - 每页数量（默认 20）

**响应**：
```json
{
    "success": true,
    "documents": [...],
    "total": 100,
    "page": 1,
    "page_size": 20
}
```

#### DELETE /api/rag/documents/{doc_id}

**描述**：删除单篇文档

### 5. 检索测试

#### POST /api/rag/search

**描述**：测试检索功能

**请求参数**：
```json

    "query": "查询语句",
    "collection_name": "资料库名称",
    "top_k": 5,
    "retrieval_mode": "hybrid"  // vector/bm25/hybrid
```

## 前端页面说明

### 页面结构

前端是单页应用（SPA），包含以下功能区域：

#### 1. 智能对话页面（顶部）

- **资料库选择下拉框**：实时从后端获取存在的资料库列表
- **对话输入框**：支持文本输入和发送
- **对话历史区域**：Markdown 渲染，支持代码高亮和公式
- **参数配置面板**：温度、最大 token、检索数量等

#### 2. 上传商品资料页面

- **资料库选择下拉框**：从后端获取存在的资料库列表
- **新建资料库按钮**：弹窗输入新库名称
- **文件上传区域**：支持多文件上传
- **文本粘贴区域**：支持直接粘贴文本内容
- **上传状态提示**：实时显示上传进度和结果

#### 3. 商品资料管理页面

- **资料库选择下拉框**：从后端获取存在的资料库列表
- **清空资料库按钮**：选中资料库后可点击，二次确认
- **资料库统计信息**：文档数量、切片数量、检索模式

#### 4. 商品资料列表页面

- **资料库选择下拉框**：从后端获取存在的资料库列表
- **文档列表表格**：展示文档 ID、来源、大小、时间
- **分页控件**：支持分页浏览
- **删除操作**：支持单篇文档删除

### 核心交互流程

#### 上传文档流程
```
1. 用户选择目标资料库（下拉框）
2. 点击「新建资料库」（如需新建）
3. 弹窗输入新库名称 → 确认创建
4. 选择文件或粘贴文本
5. 点击「上传」按钮
6. 后端处理：文件解析 → 文本切片 → 向量化 → 入库
7. 上传成功 → 刷新所有页面的资料库下拉列表
```

#### 清空资料库流程
```
1. 用户从下拉框选中要清空的资料库
2. 「清空」按钮变为可点击状态
3. 点击按钮 → 二次确认弹窗
4. 后端删除 collection 所有文档并移除该库
5. 所有页面下拉列表自动刷新，该库消失
6. 显示成功提示
```

#### 对话流程
```
1. 用户选择资料库（下拉框）
2. 输入问题并发送
3. 后端：查询优化 → 混合检索 → 重排序 → 生成回答
4. 流式返回结果到前端
5. 前端实时渲染 Markdown
```

## 常见问题

### Q1: 文档上传成功但向量库无数据？

**可能原因**：
- ChromaDB 的 metadata 必须是非空字典
- 嵌入模型调用失败
- 文档内容为空

**解决方法**：检查后端日志，确认 `add_documents` 方法的执行情况。

### Q2: 检索结果不准确？

**优化建议**：
- 调整 `CHUNK_SIZE` 和 `CHUNK_OVERLAP` 参数
- 调整 `SIMILARITY_THRESHOLD` 阈值
- 使用混合检索模式（`hybrid`）
- 启用查询优化和重排序

### Q3: API 调用超时？

**解决方法**：
- 增加 `REQUEST_TIMEOUT` 配置
- 检查网络连接
- 确认 API 密钥有效且余额充足
